Internship AI Platform

Complete Technical Documentation & Interview Guide

1. System Overview

The Internship AI Platform is a modern recruitment tool that leverages Generative AI and Vector Search to automate resume analysis and job matching. It is designed with a service-oriented architecture, splitting business logic from AI-intensive processing.

Java Spring Boot Python FastAPI React Groq AI Pinecone DB MySQL Tesseract OCR

2. Architecture

A distributed system consisting of three main layers:

Frontend (Client)

Built with React and Tailwind CSS. It handles user interactions, file uploads, and state management via context hooks.

Backend (Orchestrator)

Spring Boot application that manages the database, user sessions, and security. It acts as a gateway to the AI services.

AI Engine (Processing)

FastAPI service that performs OCR, LLM-based parsing, and vector search. It interacts with Groq for intelligence and Pinecone for similarity search.

3. Detailed Workflows

A. Resume Analysis & Matching

Step 1: User uploads a PDF. Frontend sends it to Spring Boot /upload.

Step 2: Spring Boot forwards the file to FastAPI /analyze-resume.

Step 3: FastAPI uses pytesseract if the PDF is scanned text. If it is high-quality PDF, PyPDF2 is used.

Step 4: Extracted text is sent to Groq AI. A structured JSON is returned containing skills, education, and experience.

Step 5: The structured data is embedded into a 384-dimensional vector and queried against Pinecone.

Step 6: Matches are returned to the user with a relevance score.

B. Job Creation Workflow

1. Input: Admin posts a job via /create-job.

2. Persistent Storage: Job is saved in MySQL for general display.

3. AI Indexing: Job details are sent to FastAPI to generate a "semantic query" and stored in Pinecone for future search matching.

4. Backend API Reference

UserController

  • POST /register: New user signup.
  • POST /login: Authentication (Session-based).
  • GET /get-profile: Fetches user metadata.

ProfileController

  • POST /upload: Multipart/form-data upload for resumes.
  • POST /add-profile: Saves user phone, gender, and regional data.

5. Interview Preparation Guide

Typical Question: "How do you handle scanned PDFs?"

Model Answer: "The system uses a hybrid approach. First, we try to extract text directly using PyPDF2. If the resulting text is too short or empty, we assume it's a scanned image and use Tesseract OCR through the pdf2image and pytesseract libraries to extract text content."

Typical Question: "Why use Pinecone and Vector Search?"

Model Answer: "Keyword matching is limited. Vector search allows us to find 'related' jobs. For example, if a resume says 'Java Developer' and a job says 'Spring Boot Engineer', semantic search understands they are related even if the keywords don't match exactly."

Typical Question: "How did you manage authentication?"

Model Answer: "I implemented a session-based authentication using Spring Security. The frontend uses a React Context (AuthContext) to track the logged-in state and persists the user session across page reloads by checking the backend /me endpoint on initialization."
